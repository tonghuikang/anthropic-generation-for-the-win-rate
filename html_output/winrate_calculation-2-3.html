2 How is AI revolutionizing everyday tasks in the data science industry?

Tong Hui Kang

You no longer need to memorize all the complex SQL keywords.
For example, you have this task
given a table of uid, transition_unix_time, calculate the intervals

There are chatbots that are able to translate your human language query into working SQL code.
WITH transitions (uid, transition_unix_time) AS (
VALUES
(1, 1610000000),
(1, 1610003600),
(1, 1610007200),
(2, 1610000800),
(2, 1610004400),
(3, 1610005000)
)
SELECT
uid,
transition_unix_time,
transition_unix_time - LAG(transition_unix_time) OVER (PARTITION BY uid ORDER BY transition_unix_time) AS interval
FROM transitions
ORDER BY uid, transition_unix_time
LIMIT 10

The chatbot is also able to run the code and verifies that it works. You can also ask for follow-ups, like
include more data in the mock table, calculate the standard deviation for each user and filter intervals that are greater than a standard deviation

When you are happy with the result of the chatbot, you can then copy over the query, edit it, and use the edited query on the actual business data.
In the future, you might not even need to edit it. The AI might actually be able to directly access and query the actual business data and directly give you the insights.		 <questions>

5 How good is Python for competitive programming?

Tong Hui Kang
Machine Learning Engineer in Recommendations
I agree that you should not use Python if you are doing competitive programming competitively.
These are the situations where you should not learn Python
IOI and related contests (Only C++ is allowed for IOI 2022)
ICPC and related contests (good luck finding good teammates who also use Python)
Even if you are doing competitive programming non-competitively, these are the obstacles that you may face
Problems where the time constraints are tight. Sometimes, the author intends to disallow O(n3) solutions but allow O(n2 log n) solutions. Your Python solution, even though implemented in O(n2 log n), may be evaluated as Time Limit Exceeded.
Problems with strong memory constraints. Some problems on Codeforces impose unusual memory constraints as little as 32MB. These problems may require the use of variables with a small bit length (like C++ short), which is not a feature of Python.
Python grandmasters are few and far in between. There has only been one person who uses only Python, never C++, and reached Grandmaster.
However, Python is actually a good language for non-competitive competitive programming.
Most of the problems that solution will fail because you do not have a correct algorithm, not because of Python implementation.
When you start out competitive programming, you want to learn to craft and implement algorithms fast. Python code is less verbose, and concepts can be seen more clearly.
C++ is not that friendly to beginners - for example you need understand and juggle types and pointers. The same points that discusses what should be your first programming language also apply here.
The most recent problem that Python gave me a bunch of TLE is Question 4 of a recent Google Kick Start problem.
https://codingcompetitions.withg...
There are also a few ways you can reduce the occurrence of TLE with Python code
Use of PyPy3, which is a faster implementation of Python. Most competitive programming website has PyPy3 as a submission option. You do not need to change how you write the code, but usually implementations of PyPy3 do not implement libraries like numpy.
Avoid calling functions recursively - see this comment for an explanation and a workaround. I use a different workaround - I defined a function to call when I enter a node, and another function to call when I exit a node, and track the progress of the depth first search with a hashmap and stack. You can see how I implement it in the linked solution from Kick Start.
Avoid accessing list of lists. Probably you can use math and divmod to compute which indices should you access in the flattened matrix.
Libraries. C++ and Java has binary search trees built in, but not Python. I actually extracted a copy of the sortedcontainers library for use in contests.
Input output templates. I append input = sys.stdin.readline at the start of the code to read input faster. There is another template that is being used that is probably even faster.		 <questions>

9 You have a classifier with 100% precision. You have another classifier with 100% recall. What happens when you combine them?

Tong Hui Kang
Machine Learning Engineer in Recommendations
This is what a classifier with 100% precision actually means
A classifier with 100% precision means that all the instances it identifies as positive are actually positive. In other words, there are no false positives.
A classifier that correctly surfaces 1 true positive example among 10,000 items with 100 actual positive has 100% precision. In this example, the recall is 1%.
This is what a classifier with 100% recall actually means
A classifier with 100% recall means that it identifies all the actual positive instances as positive. In other words, there are no false negatives.
A classifier that correctly suggests all 10,000 as positive among 10,000 items with 100 actual positive has 100% recall. In this example, its precision is 1%.
The only guarantee of the combined classifier is that it has 100% precision. Again, that could mean that the classifier could only surface one true positive example among ten thousand items with a hundred actual positives. Its recall can be very bad, making it useless.		 <questions>

Unlike previous models published by Stability AI, SD3 and SD3-Turbo were not open-weights models when they were published.
As of writing, SD3 costs 6.5 cents per image, whereas SD3-Turbo costs 4 cents per image.[3] This is competitive with DALL·E 3 at 4 cents per image, which has a similar performance.[4]
This article [5] from Stability AI describes how they distilled SD3-Turbo from SD3. In the abstract, the article states that
We introduce Latent Adversarial Diffusion Distillation (LADD), a novel distillation approach overcoming the limitations of ADD.
We apply LADD to Stable Diffusion 3 (8B) to obtain SD3-Turbo, a fast model that matches the performance of state-of-the-art text-to-image generators using only four unguided sampling steps.

In summary, SD3-Turbo is a distillation of SD3. In layperson language, SD3-Turbo is a smaller model (which is cheaper and faster to run) compared to SD3. The performance of SD3-Turbo might be slightly worse than SD3.
